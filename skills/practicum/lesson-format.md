# Формат уроку Vibe Typing — спільне ядро

Спільні правила генерації уроку для всіх навичок плагіна
(practicum, how-to, technology). Викликаюча навичка дає: відібрані
фрагменти коду та ІМ'Я файла уроку.

## Мова уроку

- Усі тексти уроку (title, explanation, notes, тексти курсу і чату) —
  РІДНОЮ мовою користувача: мовою, якою він спілкується в сесії
  (українською → українською, 中文 → китайською, англійською →
  англійською). Якщо мову визначити неможливо — англійською.
- UI темплейта (кнопки, результати, панель дерева) — англійською;
  його НЕ локалізуй і не редагуй при збиранні.

## Обмеження коду фрагментів

- Таби заміни на 4 пробіли. Трейлінг-пробілів бути не повинно.
- Лише символи, які користувач може набрати на своїй клавіатурі: `\n`,
  друкований ASCII (U+0020–U+007E) та літери рідної мови користувача
  (для української — кирилиця U+0400–U+04FF). Друкарську пунктуацію
  заміни на ASCII-аналоги: `—` `–` → `-`, `“` `”` `«` `»` → `"`,
  `‘` `’` → `'`, `…` → `...`, нерозривний пробіл → звичайний.
  Emoji та інші спецсимволи — прибери або перепиши без них.
- Сумарно на урок: до 4000 символів коду — жорстка межа (~20 хв друку);
  орієнтир знизу ~2500, але краще менший цілісний урок, ніж штучно
  склеєний з незв'язаних шматків.
- Фрагмент може бути частиною файла — познач це в полі `file`:
  «шлях (рядки 10–40)» або «шлях (2/6)».

## Що не друкується (skips)

- Коментарі та докстрінги користувач НЕ друкує: познач їх діапазонами
  в полі `skips` фрагмента — двигун покаже їх іншим кольором (курсивом)
  і курсор пролетить їх автоматично. Інфу вони дають, друк — ні.
- Рядковий коментар (рядок цілком) — діапазон від початку рядка до
  початку наступного (включно з `\n`); сусідні такі рядки об'єднуй
  в один діапазон. Inline-коментар у кінці рядка — від пробілів перед
  ним до кінця коментаря, БЕЗ `\n` (Enter користувач друкує сам).
- Багатослівні коментарі, що не додають розуміння, краще взагалі
  не включати в `code` (скороти фрагмент до ядра).
- Діапазони не повинні покривати фрагмент цілком — має лишатись
  що друкувати.

## Дерево проекту (tree)

- Опційне поле верхнього рівня `"tree": ["шлях/до/файла", ...]` —
  файли, збудовані ПОПЕРЕДНІМИ уроками курсу, в порядку появи.
  Двигун показує їх у дереві зліва одразу (✓), а файли поточного
  уроку додає по мірі друку.
- Шлях у `fragments[].file` ДО першої ` (` має бути точним шляхом
  файла від кореня проекту — з нього двигун будує дерево.
- Опційне `"treeTotal": N` (число) — кількість файлів УСЬОГО курсу;
  лічильник прогресу панелі рахує від нього. Без поля — від файлів,
  відомих цьому уроку.
- Разовий урок (practicum) — обидва поля не задавай.

## Пояснення фрагмента (explanation)

- Мовою користувача (див. «Мова уроку»), 2–4 речення на фрагмент,
  рівень початківця.
- Відповідай на два питання: «що це» і «навіщо саме тут у проєкті».
- Терміни без розшифровки не вживай: назвав термін — одразу поясни його
  простими словами.

## Покрокові пояснення (steps)

- Розбий код фрагмента на логічні блоки по 1–5 рядків: оголошення,
  ініціалізація, умова, цикл, повернення тощо. Блоки йдуть підряд
  і разом покривають увесь фрагмент.
- Для кожного блоку — `note`: 1–2 речення «як дитині»: що ми зараз
  пишемо і навіщо воно тут. Без жаргону; якщо термін потрібен —
  одразу поясни його простими словами. Нотатка з'являється поруч із
  кодом, поки користувач друкує саме цей блок.
- `start`/`end` — офсети блоку в `code`, вирівняні по межах рядків:
  `start` — початок рядка, `end` — початок наступного блоку
  (або кінець коду для останнього).

## lesson-JSON

```json
{
  "title": "коротка тема уроку",
  "project": "назва проєкту",
  "date": "YYYY-MM-DD",
  "tree": ["шлях/до/файла.py"],
  "fragments": [
    {
      "file": "шлях/до/файла.py",
      "explanation": "2–4 речення мовою користувача",
      "code": "повний текст фрагмента з \\n",
      "skips": [
        { "start": 0, "end": 0 }
      ],
      "steps": [
        { "start": 0, "end": 0,
          "note": "1–2 речення: що пишемо в цьому блоці і навіщо" }
      ]
    }
  ]
}
```

`start`/`end` — офсети в `code`: блок — це `code[start:end]`.
`skips` — опційне: діапазони, які не друкуються (див. вище).

## Перевірка офсетів (обов'язково)

Збережи JSON у тимчасовий файл і виконай:

```bash
python3 - <<'EOF'
import json
lesson = json.load(open('/tmp/practicum-lesson.json'))
tree = lesson.get('tree', [])
assert len(tree) == len(set(tree)), 'tree: дублікати шляхів'
assert all(isinstance(p, str) and p.strip() and ' (' not in p
           for p in tree), 'tree: некоректний шлях'
tt = lesson.get('treeTotal')
assert tt is None or (isinstance(tt, int) and tt > 0), 'treeTotal: додатне число'
for f in lesson['fragments']:
    code = f['code']
    assert '\t' not in code, 'таби мають бути замінені пробілами'
    # 'Ѐ'..'ӿ' — кирилиця; для іншої рідної мови користувача заміни
    # діапазон на відповідний (напр. CJK U+4E00–U+9FFF)
    bad = {c for c in code
           if not (c == '\n' or ' ' <= c <= '~' or 'Ѐ' <= c <= 'ӿ')}
    assert not bad, f'символи, яких нема на клавіатурі: {bad!r}'
    assert all(l == l.rstrip() for l in code.split('\n')), 'трейлінг-пробіли'
    skips = sorted(f.get('skips', []), key=lambda s: s['start'])
    for s in skips:
        assert 0 <= s['start'] < s['end'] <= len(code), 'skip поза межами'
    for a, b in zip(skips, skips[1:]):
        assert a['end'] <= b['start'], 'skips перетинаються'
    assert sum(s['end'] - s['start'] for s in skips) < len(code), \
        'фрагмент не може бути skip цілком'
    prev_end = 0
    for s in f['steps']:
        assert 0 <= s['start'] < s['end'] <= len(code), 'офсети поза межами'
        assert s['start'] == prev_end, 'кроки мають іти підряд без дірок і перетинів'
        assert s['start'] == 0 or code[s['start'] - 1] == '\n', \
            'крок має починатись з початку рядка'
        assert s['note'].strip(), 'порожня нотатка'
        prev_end = s['end']
    assert prev_end == len(code), 'кроки мають покривати весь фрагмент'
print('offsets ok')
EOF
```

Якщо `offsets ok` нема — виправ офсети й повтори. Без цієї перевірки
HTML не збирай.

## Збирання HTML

1. Прочитай `template.html` — він лежить ПОРЯД із цим файлом
   (`skills/practicum/`).
2. Заміни рядок `__LESSON_JSON__` на вміст lesson-JSON, попередньо
   замінивши в ньому `</` на `<\/` (валідний JSON-escape; без цього
   `</script>` у коді фрагмента обірве скрипт-блок сторінки).
3. Збережи у `.practicum/lessons/<ім'я від викликаючої навички>.html`
   (директорію створи).
4. Якщо `.practicum/` нема в `.gitignore` проєкту — запропонуй
   користувачу додати (не додавай сам).
5. Відкрий: `xdg-open <файл>` і скажи користувачу, що урок готовий.

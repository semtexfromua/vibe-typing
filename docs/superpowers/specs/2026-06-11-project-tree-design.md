# Дизайн: живе дерево проекту в уроці

Дата: 2026-06-11. Статус: затверджено користувачем.

Гейміфікація і контекст: панель зліва з деревом проекту, яке росте
по мірі друку. Користувач бачить, де він у проекті і скільки вже
збудовано курсом.

## Панель

- Фіксована зліва від центрованої колонки уроку, моноширинний шрифт,
  ієрархія відступами, корінь — назва проекту (`LESSON.project`).
- Показується лише коли лівого поля достатньо (~300px+),
  на вузьких екранах прихована (нічого не ламає, друк не залежить
  від неї).

## Поведінка

- Дерево показує ТІЛЬКИ файли курсу/уроку (скелет, який будує
  користувач), не весь реальний проект.
- Файли з `tree` (збудовані попередніми уроками) — видимі одразу,
  спокійним кольором, з ✓.
- Файл поточного фрагмента: з'являється (разом з новими папками
  на шляху) при вході у фрагмент, з анімацією, підсвічений акцентом —
  «пишемо зараз». Нова папка анонсується рядком `+ mkdir <шлях>`
  (анімований рядок у дереві, НЕ друкований текст).
- Фрагмент завершено → файл стає «готовим» (✓, спокійний колір).
- Навігація назад: файл лишається в дереві (раз з'явився — росте
  тільки вперед у межах сесії уроку).

## Дані

- lesson-JSON, нове опційне поле верхнього рівня:
  `"tree": ["шлях/файл1", "шлях/файл2"]` — файли, збудовані
  попередніми уроками курсу, у порядку появи.
- Файли поточного уроку: з `fragments[].file`, шлях — частина до
  першої ` (` (суфікси «(рядки X–Y)», «(2/6)» відкидаються).
- Папки виводяться зі шляхів, окремо не описуються.
- practicum (разовий урок): `tree` відсутнє → дерево лише з файлів
  уроку.
- how-to/technology: генерують `tree` накопичувально; для цього рядки
  уроків у course.md МУСЯТЬ містити точні шляхи файлів уроку.

## Зміни

- `template.html`: панель, рендер дерева, поява/статуси вузлів.
- `lesson-format.md`: поле `tree`, правило точних шляхів у course.md,
  перевірка у валідаторі (tree — список рядків, без дублів).
- `how-to/SKILL.md` (і майбутня technology): інструкція генерувати
  `tree` з course.md.
- Існуючі уроки ai-post-generated-bot перезібрати з tree.

## Поза скоупом

- Повне дерево реального проекту, згортання папок, кліки по вузлах.
- Збереження стану дерева між уроками поза embedded-даними.

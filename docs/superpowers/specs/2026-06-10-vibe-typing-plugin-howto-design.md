# Vibe Typing — плагін + команда /how-to (курс по існуючому проекту)

**Дата:** 2026-06-10
**Статус:** дизайн схвалено
**Базується на:** `2026-06-10-practicum-typing-design.md` (Practicum v2 реалізований)

## Проблема

`/practicum` вчить лише на свіжому diff. Хочеться навчитись писати весь свій
існуючий проект з нуля: команда, що аналізує проект, будує педагогічно
впорядкований план («як би ми писали це з нуля») і веде через цикл
typing-уроків того ж формату. Плюс інструмент переростає одну команду —
потрібна упаковка як плагін «vibe-typing» з родиною команд.

## Рішення (один абзац)

Репо стає Claude Code-плагіном **vibe-typing** з двома командами:
`/vibe-typing:practicum` (як було) і `/vibe-typing:how-to` (нова). Перший
виклик `/how-to` аналізує проект, показує план курсу + знахідки поганих
практик, після підтвердження записує стан у `.practicum/course.md` і генерує
урок 1. Кожен наступний виклик генерує перший невідмічений урок і ставить
`[x]`. Спільні правила генерації уроку виносяться в `lesson-format.md`,
template.html один на обидві команди.

## Зафіксовані вимоги

| Питання | Рішення |
|---|---|
| Назва/упаковка | Плагін `vibe-typing`; команди намespace-яться: `/vibe-typing:practicum`, `/vibe-typing:how-to` (свідомо прийнято) |
| Цикл курсу | Одна stateful-команда; стан — `.practicum/course.md` з чекбоксами |
| Зміст уроку | Реальний код проекту: малий файл → один урок, великий → кілька уроків поспіль; однотипне (10 ендпоінтів) → 1–2 представники типу; патерн (generic repository) → сам патерн + використання; моделі/схеми — повністю |
| Погані практики | Агент знаходить і показує РАЗОМ із планом («тут X, сучасно — Y»); користувач до старту вирішує: вчимо як є / спершу виправити; нотатки уроків чесно згадують кращу практику |
| Обсяг уроку | Той самий ліміт: 2500–4000 символів ≈ 10–20 хв |

## Архітектура (структура плагіна)

Репо `~/Programming/practicum` перейменовується на `~/Programming/vibe-typing`:

```
vibe-typing/
├── .claude-plugin/
│   ├── plugin.json          { "name": "vibe-typing", "version": "0.2.0", ... }
│   └── marketplace.json     локальний маркетплейс для установки
├── commands/
│   ├── practicum.md         (як було)
│   └── how-to.md            (нова)
├── skills/
│   ├── practicum/
│   │   ├── SKILL.md         збір diff + відбір фрагментів → lesson-format.md
│   │   ├── lesson-format.md СПІЛЬНЕ ЯДРО (див. нижче)
│   │   └── template.html    (без змін)
│   └── how-to/
│       └── SKILL.md         аналіз + план + стан курсу → lesson-format.md
├── sample-lesson.json, README.md, docs/
```

- **Установка:** `/plugin marketplace add ~/Programming/vibe-typing` +
  установка плагіна; старі симлінки `~/.claude/commands/practicum.md` і
  `~/.claude/skills/practicum` видаляються. Точні команди установки
  верифікуються на етапі реалізації (критерій: `/vibe-typing:practicum`
  з'являється і працює).
- Зміни SKILL.md підхоплюються живо; маніфест — через `/reload-plugins`.

## Спільне ядро: `skills/practicum/lesson-format.md`

Виноситься з practicum/SKILL.md без зміни семантики:

- формат lesson-JSON (title/project/date/fragments: file, explanation, code,
  steps[{start,end,note}]);
- правила кроків і нотаток «як дитині» (блоки 1–5 рядків, суцільне покриття,
  вирівнювання по рядках);
- обмеження коду (таби → 4 пробіли, без trailing-пробілів, без символів
  поза BMP);
- python-скрипт валідації офсетів (обов'язковий перед збиранням);
- збирання HTML: читання template.html з директорії навички practicum,
  заміна `__LESSON_JSON__` з escape `</` → `<\/`, збереження в
  `.practicum/lessons/`, `xdg-open`, пропозиція `.gitignore`.

Обидві SKILL.md посилаються: «прочитай lesson-format.md поряд із цією
навичкою» (для how-to — `../practicum/lesson-format.md`).

## Логіка /how-to

**Аргумент (опціонально):** фокус-шлях — будувати курс лише по підсистемі
(`/vibe-typing:how-to src/api`).

**Режим 1 — нема `.practicum/course.md`:**

1. Аналіз проекту: структура, точки входу, залежності модулів, стек.
   Не git-залежний (працює і без репо); порожній/нечитний проект →
   повідомити і зупинитись.
2. Побудова плану курсу:
   - топологічний порядок: фундамент → залежності → верхівка (приклад
     FastAPI: .env+settings → db → main/lifespan → моделі → схеми → роутери);
   - модуль вчимо ПІСЛЯ того, від чого він залежить;
   - правила відбору змісту — з таблиці вимог (представники однотипного,
     патерн+використання, моделі/схеми повністю);
   - кожен урок ≤ 2500–4000 символів; великий файл → кілька уроків поспіль
     («db.py: двигун», «db.py: сесії») — рішення приймається на етапі плану;
   - орієнтир розміру курсу: 5–25 уроків.
3. Пошук поганих/застарілих практик → секція «Знахідки».
4. Показати користувачу план + знахідки в чаті. ДОЧЕКАТИСЬ рішення:
   «вчимо як є» / «спершу виправ X» (виправлення — окрема робота поза
   /how-to; після неї курс будується по виправленому коду).
5. Записати `.practicum/course.md`, згенерувати урок 1 (через
   lesson-format.md), відмітити `[x]`.

**Режим 2 — є `.practicum/course.md`:**

1. Розпарсити чекбокси; перший `- [ ]` — наступний урок.
2. Прочитати АКТУАЛЬНИЙ код файлів уроку (проект міг змінитись після плану).
3. Згенерувати урок → `.practicum/lessons/course-NN-<slug>.html` → `[x]`.
4. Всі `[x]` → повідомити «курс завершено»; новий курс = видалити course.md
   (підказати це користувачу).

## Формат course.md

```markdown
# Курс: <назва проекту> — <одне речення, про що він>

Фокус: <шлях або «весь проект»>. Створено: YYYY-MM-DD.

## Знахідки (рішення: вчимо як є)

- ⚠ settings.py: pydantic v1 BaseSettings — сучасно pydantic-settings v2.

## Уроки

- [x] 01 · .env + settings.py — конфіг через pydantic-settings
- [ ] 02 · db.py — async engine і фабрика сесій
- [ ] 03 · main.py — FastAPI app + lifespan
```

Рядок уроку: `- [ ] NN · <файли/тема> — <що вчимо, один рядок>`.
Файл редагований руками: можна міняти порядок, видаляти, додавати уроки —
агент лише шукає перший `- [ ]` і вірить файлу.

## Помилки і межі

- Порожній проект / нема читних файлів коду → повідомити, курс не будувати.
- course.md зіпсований (нема секції «Уроки» чи жодного чекбокса) →
  повідомити і запропонувати видалити файл.
- Урок посилається на файл, якого вже нема → пропустити урок, відмітити
  `[x]` з приміткою `(файл зник)`, перейти до наступного.
- Template.html та формат уроку НЕ змінюються — це чисто генераційна фіча.

## Тестування і критерій успіху

- Установка: після міграції в плагін `/vibe-typing:practicum` працює як
  раніше (перевірка на golden-уроці), симлінки видалені.
- `/vibe-typing:how-to` на реальному FastAPI-проекті: будує осмислений план
  зі знахідками → після підтвердження урок 1 проходиться до метрик →
  повторний виклик генерує урок 2 і відмічає `[x]` у course.md.
- Ручна перевірка course.md: відредагувати руками порядок — наступний виклик
  поважає зміни.

## Поза межами (свідомо)

- Публікація плагіна в публічний маркетплейс.
- Автовизначення «урок пройдено» (зараз `[x]` = згенеровано; проходження —
  на совісті користувача).
- Прогрес/статистика курсу поверх localStorage уроків.
- Нові команди родини vibe-typing (буде окремими спеками).
